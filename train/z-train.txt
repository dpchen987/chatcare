0 首先在加载模型的时候，我们增加一个BitsAndBytesConfig的参数：
0 我们看一下重点的参数BitsAndBytesConfig
0 load_in_4bit：通过替换bitsandbytes中的Linear层为FP4/NF4层，启用4位量化
0 bnb_4bit_compute_dtype：设置计算类型，它可能与输入时的类型不同。例如，输入可能是fp32，但计算可以设置为bf16以获得速度提升。
0 bnb_4bit_use_double_quant：这个标志用于是否开启二次量化
0 bnb_4bit_quant_type：有两个参数可以选fp4和nf4，默认是fp4
0 llm_int8_threshold：这与LLM.int8() : 8-bit Matrix Multiplication for Transformers at Scale论文中描述的异常值检测的异常值阈值相对应。论文链接：https://arxiv.org/abs/2208.07339 任何超过这个阈值的隐藏状态值都将被视为异常值，并且对这些值进行的操作将在fp16中进行。通常，这些值符合正态分布，即大多数值位于[-3.5，3.5]范围内，但对于大型模型，有一些例外的系统异常值分布非常不同。这些异常值通常在区间[-60，-6]或[6，60]之间。对于数值幅度约为5左右的值，int8量化效果良好，但超过此范围后，会导致显著的性能损耗。一个好的默认阈值是6，但对于更不稳定的模型（小型模型、微调模型），可能需要一个较低的阈值。
